NMEA sentence (ASCII, line‑oriented)

File/wire layout (single sentence, terminated by CRLF):
  <start>$ (1 byte, literal 0x24)
  <talker_id> (2 ASCII chars, e.g. "GP", "GL")
  <sentence_id> (3 ASCII chars, e.g. "GGA", "RMC")
  <field0> (ASCII, may be empty)   // fields are comma separated
  <field1>
  ...
  <fieldN>
  *<checksum> (2 hex digits, uppercase A‑F or 0‑9)
  <CR><LF> (2 bytes, 0x0D 0x0A)

General rules:
  - The number of fields depends on <sentence_id>.  The library defines a fixed order for each supported ID (GBS, GGA, GLL, GSA, GST, GSV, RMC, VTG, ZDA).
  - Each field is an ASCII token; empty token means missing value.
  - Numeric fields are either integer decimal (e.g. satellite count) or fractional "value/scale" encoded as a decimal string (e.g. latitude "ddmm.mmmm").
  - Direction fields are a single character (N,S,E,W) and are parsed as int sign.
  - Boolean "valid" fields are a single character ('A' for active, 'V' for void) and parsed to bool.
  - Enum fields (status, mode, FAA mode, GSA mode, GSA fix) are a single ASCII character with defined set; any other character is treated as unknown.
  - The checksum byte is computed as XOR of all characters between '$' and '*', exclusive.

Concrete per‑sentence field layout (comma‑separated, no surrounding spaces):

GBS (Garmin Bad Signal):
  $<talker>GBS,<time>,<err_lat>,<err_lon>,<err_alt>,<svid>,<prob>,<bias>,<stddev>*hh\r\n
  Fields:
    0 time            – hhmmss.sss (fractional, parsed as minmea_time)
    1 err_latitude    – ddmm.mmmm (fractional, parsed as minmea_float)
    2 err_longitude   – dddmm.mmmm (fractional)
    3 err_altitude    – meters (fractional)
    4 svid            – integer
    5 prob            – fractional
    6 bias            – fractional
    7 stddev          – fractional

GGA (Fix data):
  $<talker>GGA,<time>,<lat>,<ns>,<lon>,<ew>,<fix>,<sat>,<hdop>,<alt>,<alt_units>,<height>,<height_units>,<dgps_age>*hh\r\n
  Fields (0‑13): time, latitude, N/S, longitude, E/W, fix_quality, satellites_tracked, hdop, altitude, altitude_units, height, height_units, dgps_age

GLL (Geographic position – latitude/longitude):
  $<talker>GLL,<lat>,<ns>,<lon>,<ew>,<time>,<status>,<mode>*hh\r\n
  Fields: latitude, N/S, longitude, E/W, time, status (A/V), mode (FAA enum)

GSA (GNSS DOP and active satellites):
  $<talker>GSA,<mode>,<fix_type>,<sat1>,...,<sat12>,<pdop>,<hdop>,<vdop>*hh\r\n
  Fields: mode (A/M), fix_type (1‑3), 12 satellite PRNs (empty if not used), pdop, hdop, vdop

GST (GNSS pseudorange error statistics):
  $<talker>GST,<time>,<rms>,<semi_major>,<semi_minor>,<orientation>,<lat_err>,<lon_err>,<alt_err>*hh\r\n
  Fields: time, rms_deviation, semi_major_deviation, semi_minor_deviation, semi_major_orientation, latitude_error_deviation, longitude_error_deviation, altitude_error_deviation

GSV (Satellites in view):
  $<talker>GSV,<total_msgs>,<msg_nr>,<total_sats>,<sat1_prn>,<sat1_elv>,<sat1_az>,<sat1_snr>,...,<sat4_prn>,<sat4_elv>,<sat4_az>,<sat4_snr>*hh\r\n
  Each message carries up to 4 satellite info blocks (prn, elevation, azimuth, SNR). Missing blocks are empty fields.

RMC (Recommended minimum data):
  $<talker>RMC,<time>,<valid>,<lat>,<ns>,<lon>,<ew>,<speed>,<course>,<date>,<variation>,<mode>*hh\r\n
  Fields: time, validity (A/V), latitude, N/S, longitude, E/W, speed over ground, course over ground, date (ddmmyy), magnetic variation, FAA mode (optional)

VTG (Track made good and ground speed):
  $<talker>VTG,<true_track>,<T>,<mag_track>,<M>,<spd_knots>,<N>,<spd_kph>,<K>,<faa_mode>*hh\r\n
  Fields: true_track_degrees, magnetic_track_degrees, speed_knots, speed_kph, FAA mode (optional). The single‑letter placeholders (T,M,N,K) are literal and must be present.

ZDA (Date & time):
  $<talker>ZDA,<time>,<day>,<month>,<year>,<hour_offset>,<minute_offset>*hh\r\n
  Fields: time, day, month, year, hour offset, minute offset

Mutator strategy:
  - Preserve leading '$' and trailing "*hh\r\n" structure; recompute checksum after any change.
  - Target <talker_id> (2 bytes) and <sentence_id> (3 bytes) to flip between known IDs and random ASCII to trigger unknown‑sentence handling.
  - For each numeric field, flip individual digits, insert overflow digits, or replace with empty token to test missing‑value paths.
  - For enum/char fields (status, mode, FAA mode, GSA mode, N/S, E/W), substitute each defined character and a few out‑of‑range characters (e.g. 'X', '\0').
  - For variable‑length fields (satellite lists in GSA/GSV), vary the count of satellite blocks by adding/removing commas, inserting extra empty fields, or duplicating blocks.
  - Preserve the number of commas required for the target sentence type; mutator may also delete a comma to test parser robustness, but must keep overall field count within a reasonable bound (≤ MAX_SENTENCE_LENGTH 80).
  - After any mutation, recalculate the XOR checksum over the bytes between '$' and '*'.  The mutator should treat the checksum bytes as dependent fields and rewrite them accordingly.
  - Do NOT exceed MINMEA_MAX_SENTENCE_LENGTH (80 bytes) – truncate or pad with spaces if needed; mutator should keep mutations small (≤ 16 bytes).

PER-MUTATION COST: keep each mutation tiny — at most ~16 bytes of write. AFL's havoc/cmplog will compose larger runs across iterations.
